# PRD — newtype-create-skill

## 目标

构建一个参考 `ex-skill` / `colleague-skill` 上传交互模式的个人人格蒸馏 skill。

重点：
- 不是深度关系分析
- 不是心理诊断
- 而是从多源聊天材料中提取“用户自己发出的消息”
- 蒸馏出一个像用户一样回复的聊天人格
- 最终生成必须尽量贴近上传记录中用户真实会说的话
- 不凭空生成无关或有害心理健康语句
- 对模型不确定的新鲜网络热词和梗，允许做轻量联网搜索辅助理解，避免误解语义

## MVP 范围

1. 3 个问题的信息录入
2. 聊天记录 / 文件 / 粘贴内容导入
3. 群聊 / 多账号场景下的“自己身份确认”机制
4. `self / other / unknown` 消息归类
5. 风格提取与回复人格生成
6. 默认简短回复的人格规则
7. 图片 / 表情 / 截图轻量使用，不做深度解读
8. 追加更新与纠错
9. 版本管理

## 识别原则

### 高优先级信号
- sender / from / nickname / is_self
- HTML class / JSON role

### 启发式信号
- 右侧通常是自己消息
- 右侧头像 / 连续右侧气泡

### 使用原则
- `self` 主学习
- `other` 作上下文
- `unknown` 降权处理

## 群聊身份确认机制

适用场景：
- 群聊导出
- 多人聊天转存
- sender / nickname 混杂的记录

处理流程：
1. 先从记录中提取候选 sender / nickname 列表
2. 让用户输入或选择“哪个名字是自己”
3. 将该名字作为 `self-name` 重新参与解析

目标：
- 降低群聊里误把别人消息当成用户自己的风险
- 提高人格蒸馏时 `self` 样本纯度

## 默认人格策略

- 生成出来的人设默认偏**短回复**
- 除非材料明确显示用户习惯长篇表达，或用户显式要求展开
- 优先保留用户原本的节奏、收尾方式和简洁度
- 优先生成与原始记录高度接近的表达，不生搬硬套人设模板
- 样本不足时宁可保守，不强行补充用户未表现过的语言习惯
- 对新鲜网络热词/梗，先理解其常见含义，再决定是否保留；仅在原始样本真实使用时才纳入人格规则

## 图片与截图处理边界

- 图片、表情包、贴纸、引用图只作为辅助信号
- 可用于判断语气、互动氛围、回复节奏
- 不对图片内容进行深度剧情解读、立场推断或心理分析
- 无法识别清楚时，降为低置信度材料

## 内容安全边界

- 不凭空推导用户的心理疾病、创伤经历或精神状态
- 不生成与材料无关的自毁、自厌、病理化、过度负面表达
- 若材料中缺少明确证据，则相关内容一律不纳入人格规则

## 新鲜热词 / 梗理解策略

- 允许在必要时使用联网搜索辅助理解最新网络热词、缩写、梗和圈内表达
- 联网搜索的目标是“避免听不懂”，不是“替用户创造更多梗”
- 搜索结果只能作为语义参考，最终是否写入人格仍以用户上传样本为准
