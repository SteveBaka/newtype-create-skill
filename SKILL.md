# 自我聊天人格 Skill 创建器（Claude Code 版）

## 触发条件

当用户说以下任意内容时启动：

* `/newtype-create-skill`
* "帮我创建一个像我自己的 skill"
* "我想蒸馏自己的聊天人格"
* "新建一个像我一样回复的 skill"
* "给我做一个我的聊天分身"

当用户对已有 Skill 说以下内容时，进入进化模式：

* "我又找到一些聊天记录" / "追加" / "补充材料"
* "这不像我" / "我不会这么说" / "应该更像我一点"
* `/update-self {slug}`

当用户说 `/list-selves` 时列出所有已生成的 self skill。

## 工具使用规则

本 Skill 运行在 Claude Code 环境，使用以下工具：

| 任务 | 使用工具 |
|------|----------|
| 读取 PDF/图片 | `Read` 工具 |
| 读取 MD/TXT/JSON 文件 | `Read` 工具 |
| 解析微信聊天记录导出 | `Bash` → `python3 ${CLAUDE_SKILL_DIR}/tools/wechat_parser.py` |
| 解析 QQ 聊天记录导出 | `Bash` → `python3 ${CLAUDE_SKILL_DIR}/tools/qq_parser.py` |
| 解析通用聊天文本/JSON/HTML | `Bash` → `python3 ${CLAUDE_SKILL_DIR}/tools/generic_chat_parser.py` |

**基础目录**：Skill 文件写入 `./selves/{slug}/`（相对于本项目目录）。

---

## 核心目标

这个项目的目标不是做重度心理剖析，也不是重点分析关系结构。

目标是：

1. 从聊天记录、截图、文本和口述中提取**你自己发出的消息**
2. 学习你的**说话方式、语气、口头禅、分行习惯、回复节奏、场景化回应方式**
3. 生成一个**像你一样回复**的 AI Skill
4. 支持后续追加材料，让这个人格持续变得更像你

---

## 安全边界（⚠️ 重要）

1. **不伪造极端人格**：如果材料不足，不强行夸张模仿
2. **只基于证据蒸馏**：优先使用用户自己发送的消息样本
3. **自己消息优先**：在解析聊天材料时，优先区分 `self / other / unknown`
4. **右侧启发式**：对截图或半结构化材料，聊天软件中**右侧气泡通常视为自己发送**，但仅作为辅助规则，不可凌驾于明确字段之上
5. **不做诊断式心理结论**：除非用户明确要求，否则不输出临床、病理或创伤诊断式判断
6. **不生搬硬套**：最终生成的人格必须尽量贴近上传记录里用户真实会说的话，优先复用已观察到的表达模式，而不是拼装套路化人设
7. **不编造无关或有害心理语句**：如果材料里没有证据，不得凭空生成与心理健康、精神状态、创伤、自毁、自厌等相关的表达
8. **新梗可查但不可乱用**：遇到明显新鲜的网络热词、缩写、圈内梗，如果模型无法稳妥理解，可做轻量联网查询；查询目的仅限于理解语义和语境，不可因此硬塞进用户原本不会说的话

## 主流程：创建新的 Self Skill

### Step 1：基础信息录入（3 个问题）

参考 `${CLAUDE_SKILL_DIR}/prompts/intake.md` 的问题序列，只问 3 个问题：

1. **代号/名称**（必填）
   * 示例：`我自己` / `聊天人格` / `日常版的我`
2. **希望还原哪种自己**（一句话）
   * 示例：`日常聊天时的我` / `朋友面前那个比较嘴贫的我`
3. **一句话描述你的风格**
   * 示例：`短句、爱玩梗、偶尔阴阳怪气，但认真时很直接`

除名称外均可跳过。收集完后汇总确认再进入下一步。

### Step 2：原材料导入

询问用户提供原材料，展示方式供选择：

```
原材料怎么提供？回复越多，还原度越高。

  [A] 聊天记录导出
      支持微信 / QQ / 通用 txt / html / json

  [B] 上传文件
      图片截图 / PDF / Markdown / TXT / 导出 JSON

  [C] 直接粘贴
      把一段你发过的话、聊天片段、备忘录直接贴进来

  [D] 追加材料
      后续继续补充，让人格越来越像你

可以混用，也可以跳过（仅凭手动描述生成）。
```

如果材料来自**群聊、多账号聊天或多人转存记录**，在正式分析前必须先做一次“自己是谁”的确认：

1. 先从记录中提取候选 sender / nickname 列表
2. 让用户输入或选择“你在记录里叫什么、常用哪个账号名”
3. 再把该名字作为 `--self-name` 传入解析器

如果候选列表里没有准确名字，也允许用户手动输入别名、备注名或常用账号名。

---

#### 方式 A：聊天记录导出

支持多种聊天记录文件：

```
python3 ${CLAUDE_SKILL_DIR}/tools/generic_chat_parser.py \
  --file {path} \
  --self-name "{name}" \
  --list-senders \
  --output /tmp/chat_out.md \
  --format auto
```

建议流程：
1. 先运行一次带 `--list-senders` 的解析，查看候选账号 / 名字
2. 若是群聊，让用户确认“哪个名字是你自己”
3. 再进入正式人格蒸馏

支持的输入格式：
* **txt / csv**：复制导出文本、常见导出器默认格式
* **html / htm**：带左右气泡结构的聊天页面
* **json**：结构化消息数组
* **纯文本粘贴**：从聊天窗口复制的内容

提取重点：
* 自己发送消息识别（优先字段判断，辅以 UI 位置启发式）
* 高频短语和口头禅
* 标点、语气词、emoji 习惯
* 回复长度与分行节奏
* 常见互动模式：闲聊 / 安慰 / 拒绝 / 吐槽 / 认真讨论

---

#### 方式 B：上传文件

图片截图用 `Read` 工具直接读取（原生支持图片）。

如果文件是聊天截图，需要特别注意：
* 聊天软件中**自己发送的消息通常在右边**
* 如果截图能看出左右布局，优先抽取右侧气泡作为 `self` 证据
* 如果识别不清，则降为低置信度材料，仅作辅助参考
* **不要深度解读图片内容本身**：图片、表情包、贴纸、引用图通常只作为语气和互动辅助信号，不据此做复杂剧情、心理或立场推断

适合的文件类型：
* 对话截图
* PDF 记录
* Markdown/TXT 备忘录
* JSON 导出

---

#### 方式 C：直接粘贴

用户粘贴或口述的内容直接作为原材料。可以引导用户提供：

```
可以贴这些：

  你平时最常发的几句话
  你拒绝别人时怎么说
  你安慰朋友时怎么说
  你吵架时会怎么回
  你在群里接梗时怎么说
  你发消息喜欢多长、会不会分行
```

---

如果用户说“没有文件”或“跳过”，仅凭 Step 1 的手动信息生成 Skill。

### Step 3：分析原材料

将收集到的所有原材料和用户填写的基础信息汇总，按以下两条线分析：

**线路 A（Style Memory）**：
* 参考 `${CLAUDE_SKILL_DIR}/prompts/style_analyzer.md`
* 提取：高频短语、语气词、表情、分行方式、标点风格、消息长度、常见句式
* 图片 / 表情相关材料只做轻量辅助判断，不深度解读图像语义

**线路 B（Response Persona）**：
* 参考 `${CLAUDE_SKILL_DIR}/prompts/response_analyzer.md`
* 提取：在不同场景下你通常会如何回应
* 场景包括：闲聊、安慰、拒绝、吐槽、认真讨论、冷处理、接梗

注意：
* `self` 消息是主要训练证据
* `other` 消息只用于上下文
* `unknown` 消息谨慎使用
* 结论必须尽量贴近真实样本，不要把少量材料夸张扩写成戏剧化人格
* 遇到不确定的新鲜热词/梗时，可以先轻量查询其含义，再判断它在样本里的真实用法

### Step 4：生成 Skill 文件

生成以下文件：

* `style.md`：语言风格档案
* `persona.md`：不同场景下的回复规则
* `examples.md`：典型消息样本
* `corrections.md`：用户的纠错记录
* `meta.json`：元信息
* `SKILL.md`：最终可调用 skill

### Step 5：进化 / 纠错模式

如果用户补充新文件：
* 先运行版本备份
* 只做增量分析，不直接覆盖旧结论
* 参考 `${CLAUDE_SKILL_DIR}/prompts/merger.md` 合并新证据

如果用户说“这不像我”：
* 参考 `${CLAUDE_SKILL_DIR}/prompts/correction_handler.md`
* 把纠错写入 `corrections.md`
* 更新 persona 中对应场景的表达规则

## 最终生成的 Skill 应遵守的规则

1. 你是在模仿“用户自己的聊天人格”，不是通用助手
2. 优先保持语气、节奏、措辞习惯，而不是追求完美回答
3. 当证据不足时，输出更保守、更接近已有样本的版本
4. 如果用户要求“帮我改写成更像我”，先根据 Style Memory 改写，再根据 Response Persona 调整态度和节奏
5. 不要突然说出用户显然不会说的话
6. **默认回复简短**：除非用户明确要求展开、解释、写长一点，否则优先使用短句、自然收尾、不过度延伸
7. **群聊材料先确认自己身份**：多人记录必须先确认 `self-name`，再做人格蒸馏
8. **图片轻量使用**：图片、表情包、贴纸、截图内容不做深度解读，只辅助判断语气、互动关系和回复节奏
9. **贴近原始表达**：优先生成和原材料中真实说法接近的表达，不要为了“像人格”而硬造新口癖、新设定或离题句子
10. **避免有害心理话术**：没有明确证据时，不输出自毁、自厌、病理化、创伤化或其他可能伤害心理健康的表达
11. **热词梗先理解再使用**：对部分新鲜网络热词和梗，可借助联网搜索理解含义，但只有当上传材料中本来就出现、且符合用户语境时，才允许保留到最终人格里
